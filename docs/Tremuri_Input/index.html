Read the Tremuri Input File

Reading a Tremuri file as input is one of the primary options to generate the 3D model of a masonry building. The procedure is performed using the file module named Tremuri_Input_func.py. Below, you can find the guideline, through which you can find how the Tremuri input file is interpreted.

Defined Functions

checkEmpty

def checkEmpty(f_lines, f_insideCounter):
    while 1:
        if not f_lines[f_insideCounter]:
            f_insideCounter = f_insideCounter + 1
        else:
            break
    return f_insideCounter

The function checks if there is a blank line when going through the lines and jumps over it to prevent errors from happening.

DataFrames

Below, you can find the DataFrames used for storing the masonry model/analysis parameters. There is a consistent nomination system for the utilized DataFrames and lists, described below:

  • [name] which stores all the data for the considered data type
  • tmp_[name] a temporary storage of the data for each of the individual items
  • empty_tmp_[name] This is an empty version of the tmp_[name]. As some of the fields are not mandatory, it may result in a problem where a value of previous item remains in the DataFrame as there is no respective value in the new item to replace it. Therefore, this version is used to clear all the existing values in the DataFrame, so no traces of previous item values is remained.
  • tmp_[name]_list a list which stores all the individual items and at the very end is converted to the [name] DataFrame

wall

Stores all the data for the walls. Data is stored in the format of:

- No

The wall number

- x0

The x coordinate of the origin of the wall

- y0

The y coordinate of the origin of the wall

- angle

The angle of the wall with respect to x axis

material

Stores all the data for the utilized materials. Data is stored in the format of:

- No

The material number

- E

Modulus of elasticity of the material

- G

Shear modulus of the material

- rho

Density of the material

- fc

Compressive strength of the material

- tau0

Shear strength of the material

- verification

Maximum cohesion strength

- shear_model

Shear model of the material (0: Turnsek and cacovic; 1: Mohr-Coulomb on the effective shear area; 2: Mohr-coulomb on the gross shear area)

- drift_S

Shear ultimate drift ratio of the material (exceeding causes element failure)

- drift_F

Rocking ultimate drift ratio of the material (exceeding causes element failure)

- mu

Friction coefficient of the material

- Gc

Nonlinear deformability parameter of the material defined as: G * c

- beta

Softening parameter of the material

floor

Stores all the data for the floors. Data is stored in the format of:

- No

The floor number

- nI, nJ, nK, nL

3D nodes defining the geometry of the floor

- thickness

Floor thickness

- E1, E2

Elastic modulus of the floor in both directions

- Poisson

Floor's Poisson's ratio

- G

Shear modulus of the floor

- angle

Properties direction angle

restraint

Stores all the data for the restraints. Data is stored in the format of:

- Node

Restrained node number

- x, y, z, rx, ry

Degree of freedom to be restrained. 0: Unrestrained, 1: Restrained

floor_level

Stores all the data for the floor levels. Data is stored in the format of:

- h

Floor level

- zAxis1, zAxis2, zAxis3

Normal axis of the floor (zAxis1: parallel to X axis, zAxis2: parallel to Y axis, zAxis3: parallel to Z axis)

distrMass

Stores all the data for the distributed masses. Data is stored in the format of:

- node

Node number

- V, M

Shear (V) and bending moment (M) due to the distributed load on the elements (E) working on the node number node. These values are to be multiplied by the vertical acceleration defined in dead load statement and added to node actions coming from the analyses.

- el

Elements where the distributed load is working on

Data Interpretation and Extraction

Openening/reading the input file

To generate the masonry model, all the relevant parameters shall get extracted from the source input file.

openFile = open(completeFileLocation1, "r")  # Open the Tremuri file
inputFile = openFile.read()  # Read the Tremuri file

# read the lines of the input file
lines = inputFile.splitlines()  # split the lines of the input file
openFile.close()  # close the input file after reading its data

Find modeling parameters based on their tags

After reading the data, the lines are separated to be checked for possible modeling parameters as follows:

# go through the lines to find each of the desired sections
for lineCounter in np.arange(len(lines)):
    # find the parameters for walls
    if (("pareti" in lines[lineCounter]) or ("walls" in lines[lineCounter])) and ("!" not in lines[lineCounter]):
        # rest of the code ...

Extract data and avoid errors

During the line checks, the checkEmpty function is called to jump over the empty lines and prevent errors from happening.

Now that we have all the lines with input data, the values of each parameter, separated by SPACE are extracted:

params = lines[insideCounter].split()

In some cases, the modeling parameters are followed by a number of SPACE values that could lead to errors. Therefore, the number of parameter arguments are calculated as follows:

for paramCounter in np.arange(len(params)):
    # In some cases, the lines are followed by a number of empty values.
    # Therefore, the number of parameters are calculated to prevent any errors
    # using the extracted data, afterwards.
    if params[paramCounter] == "":
        lastItem = paramCounter - 1
        break
    elif paramCounter == len(params) - 1:
        lastItem = len(params) - 1
params = params[0:lastItem + 1]

The params variable is updated to store only the variable arguments.

Convert degree angles to radians

Then the function goes through the variables and converts the angle values presented in degrees to radian values and then stores all the variables in the tmp_[name] DataFrame.

for item in np.arange(lastItem + 1):
    # if the presented angle valules are in degrees, change them to radians
    if "°" in params[item]:
        params[item] = (float(params[item][0:len(params[item]) - 1]) / 180) * math.pi
    tmp_wall.iat[0, item] = params[item]  # store all the parameters in the temporary DataFrame

Packing all the inputs

Then all the individual items are stored in a list.

tmp_wall_list.append(tmp_wall.values.tolist()[0])  # store all the individual items in a list

Finally, after checking all the instances of a particular type, the corresponding list is converted to a DataFrame.

# convert the list of all individuals to a DataFrame
wall = pd.DataFrame(tmp_wall_list, columns=['No', 'x0', 'y0', 'angle'])

Convert the data types to relevant type

As all the data is read from a text file, their data type is string even if they represent numbers. Therefore, to be able to perform comparison and mathematical oprations, numerical data types are set as their respective types.

wall = wall.astype(float)  # convert all the variables to float
wall['No'] = wall['No'].astype(int)  # the wall numbers could be integers
wall = wall.set_index('No')  # set the wall number as the index

Store extracted data in a Microsoft Excel Workbook for further surveying

In order to store the extracted data in a Microsoft Excel workbook, the following commands can be used:

writer = pd.ExcelWriter('tremuri_input_parameters.xlsx', engine='xlsxwriter')
# Convert the dataframe to an XlsxWriter Excel object.
wall.to_excel(writer, sheet_name='wall')
